# AWS SES & Project Launch Checklist

This document covers exactly how to configure AWS Simple Email Service (SES) to send the ticket confirmations, plus the final steps you need to complete to launch the AWS SCD 2026 Ticketing Platform.

---

## Part 1: AWS SES Setup Guide

To send emails containing the user's ticket and QR code, you need to configure AWS SES and integrate it into your Node.js backend.

### 1. Verify Your Domain/Email in AWS
1. Log into the **AWS Management Console** and search for **Amazon SES**.
2. Go to **Identities** -> **Create identity**.
3. Choose **Domain** (Recommended) or **Email address**.
   - *If Domain:* You will need to add CNAME/TXT records to your DNS provider (e.g., Route53, GoDaddy) to verify ownership.
   - *If Email:* AWS will send a verification link to that email.

### 2. Move Out of SES Sandbox (CRITICAL)
By default, AWS places your SES account in a "Sandbox" where you can **only send emails to verified email addresses**. Since attendees will use arbitrary emails, you must request production access.
1. On the SES Dashboard, click **Request production access** (or "Edit your account details").
2. Fill out the form explaining your use case: *"Sending transactional event ticket confirmations and QR codes to users who purchase a pass on our website."*
3. AWS usually approves this within 24 hours.

### 3. Create IAM User Credentials
1. Go to **IAM** (Identity and Access Management) in AWS.
2. Go to **Users** -> **Create user** (name it something like `scd-ses-mailer`).
3. Attach policies directly -> Search for and attach **AmazonSESFullAccess**.
4. Once created, go to the user's **Security credentials** tab.
5. Create an **Access key** (Select "Application running outside AWS").
6. Save the **Access Key ID** and **Secret Access Key**.

### 4. Code Integration

First, install the AWS SDK and Nodemailer in your `server` directory:
```bash
cd server
npm install @aws-sdk/client-ses nodemailer
npm install -D @types/nodemailer
```

Add these to your `server/.env` file:
```env
AWS_REGION=ap-south-1  # Change to your SES region
AWS_ACCESS_KEY_ID=your_access_key
AWS_SECRET_ACCESS_KEY=your_secret_key
AWS_SES_FROM_EMAIL=tickets@yourdomain.com
```

In `server/src/features/webhook/webhookRouter.ts`, replace the `// TODO: Trigger AWS SES email` comment with your email logic. I recommend creating a utility file `server/src/shared/lib/email.ts` to handle this.

## Part 2: Final Project Launch Checklist

Here is everything you need to review before directing traffic to the site:

### 1. Payment Gateway Configuration
- [ ] **Webhook Endpoint:** Go to the Cashfree Dashboard -> Developer -> Webhooks. Add your production backend URL: `https://api.yourdomain.com/api/webhooks/cashfree`. Subscribe to `PAYMENT_SUCCESS_WEBHOOK`.
- [ ] **Secret Sync:** Ensure the webhook secret generated by Cashfree matches `CASHFREE_WEBHOOK_SECRET` in `server/.env`.
- [ ] **Environment Flip:** Ensure the frontend `getCashfree()` is pointing to production, not sandbox.

### 2. Frontend Environment Variables
Check your `c:\Users\saura\Downloads\aws-scd-2026\.env` file. Ensure it contains:
- [ ] `VITE_SUPABASE_URL`
- [ ] `VITE_SUPABASE_ANON_KEY`
- [ ] `VITE_API_URL` (Points to your backend e.g., `https://api.yourdomain.com`)

### 3. Backend Environment Variables
Check your `c:\Users\saura\Downloads\aws-scd-2026\server\.env` file.
- [ ] `PORT=3001`
- [ ] `SUPABASE_URL`
- [ ] `SUPABASE_SERVICE_KEY` (MUST be the service_role key, not the anon key, so the backend can bypass RLS for secure operations)
- [ ] `CASHFREE_APP_ID` (Production ID)
- [ ] `CASHFREE_SECRET_KEY` (Production Key)
- [ ] `CASHFREE_WEBHOOK_SECRET`
- [ ] `ADMIN_KEY` (e.g., `idkbutily`)
- [ ] `FRONTEND_URL` (e.g., `https://yourdomain.com` - used by Cashfree to redirect users back to the `/ticket/:id` page)

### 4. Database Setup
- [ ] **Create Passes:** The database has no passes by default. Log into `/admin` and create the available passes. Ensure they are toggled as **Active**.
- [ ] **Unique Constraints:** The system relies on email uniqueness. If testing manually, remember to use different emails or wait for the 2-hour pending cleanup to run.

### 5. Server Deployment
- [ ] **Production Build:** `npm run dev` (using `tsx watch`) is for development. For production, you should compile the server code:
  - Add to `server/package.json`: `"build": "tsc"` and `"start": "node dist/app.js"`
  - Or use a process manager like PM2: `pm2 start "npx tsx src/app.ts" --name scd-backend`
- [ ] **CORS Settings:** In `server/src/app.ts`, update `app.use(cors())` to restrict origins to your specific frontend URL to prevent unauthorized API calls from other domains.

### 6. Run a Live $1 Test
Before the event goes public, change one of the passes to ₹1 and do a complete purchase:
1. Buy the ticket.
2. Complete Cashfree payment.
3. Watch the frontend poll and transition to the Success Screen.
4. Open the Ticket URL and ensure the QR code renders.
5. Receive the AWS SES email.
6. Open `/scanner` on your phone, scan the QR code, and verify it plays the "Success Chime" and marks `checked_in` in the database.
